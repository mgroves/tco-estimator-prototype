Methodology & validation notes

Cost model. For each selected tool: annual cost = ($/ops-sec/yr × ops/sec) + (admin FTE + integration FTE) × engineer cost, summed across every tool you select. Couchbase is modeled the same way, but as a single consolidated platform — one admin/integration allocation instead of one per tool. That consolidation gap is the source of the savings estimate, not a discount applied to any competitor's price.

Complexity reduction. Reduction = 1 − (1 system after ÷ N systems selected today). Selecting a single tool is a 1-for-1 swap with nothing to consolidate, so it correctly shows 0%; the reduction grows as more systems are replaced by the one consolidated Couchbase platform (2 tools → 50%, 3 → 67%, 5 → 80%). It's directional — meant to make the qualitative case (less duplication, smaller footprint, fewer integration points), not a formal risk score.

Ranges, not points. Headline numbers are shown as ranges (roughly ±12–15%) because default cost-per-throughput figures are order-of-magnitude public list-price assumptions, not your contracts. Unlocking Step 03 lets you tighten the range with real numbers.

Reality check. Default $/ops-sec and FTE assumptions are meant to sit within a plausible band for list-priced enterprise data infrastructure and typical admin/integration staffing ratios — deliberately adjustable so the estimate can be validated against your own contracts and headcount before it's used in a business case.

No Couchbase discount factor. Couchbase's cost assumption is entered directly, independent of competitor pricing, to avoid implying savings that come from an assumed discount rather than real consolidation.

Pricing sources (internal only — do not publish)

Not for the public build. Kept here so anyone internal can trace where a default came from.

Conversion math. 1 sustained op/sec ≈ 31.5M ops/year. For consumption-priced services we convert their published $/million-operations rate directly: $/ops-sec/yr = blended $/M × 31.5. For node-, core-, or processor-licensed products, no vendor publishes a $/op rate at all — we take the real list price, annualize it (license amortized over 5 years + support/subscription), and divide by an assumed sustained throughput per unit, which we state explicitly. That last assumption is ours, not the vendor's, and is the main source of uncertainty below.

Firestore Standard: $0.30/M reads + $0.90/M writes, official rate card. firebase.google.com/docs/firestore/enterprise/pricing

No public list price — Ditto is quote-based only. This figure is an unverified placeholder; replace if sales has real numbers. No link.

Atlas serverless: $0.10/M reads + $1.00/M writes. mongodb.com/pricing
DynamoDB$24/ops-sec/yr
On-demand: $0.25/M read + $1.25/M write request units. aws.amazon.com/dynamodb/pricing
Redis$1/ops-sec/yr
Derived from Redis Cloud's throughput-tiered plan pricing (memory + ops/sec ceiling bundled). Wide range depending on tier — treat as a rough floor. redis.io/pricing
Memcached$0.20/ops-sec/yr
No license (open-source) — figure reflects typical managed-cache hosting cost, e.g. AWS ElastiCache. aws.amazon.com/elasticache/pricing
Oracle$30/ops-sec/yr
$47,500/processor list + 22%/yr support, amortized over 5 years, ÷ an assumed ~2,000 sustained ops/sec per licensed processor (our sizing assumption, not Oracle's). oracle.com/database/pricing
SQL Server$9/ops-sec/yr
~$7,100/core list (Enterprise) + Software Assurance, amortized, ÷ an assumed ~800 sustained ops/sec per core (our sizing assumption). microsoft.com/sql-server/pricing
Postgres$1/ops-sec/yr
No license (open-source) — figure reflects typical managed-hosting instance rates, e.g. Amazon RDS. aws.amazon.com/rds/postgresql/pricing
MongoDB (Vector)$24/ops-sec/yr
MongoDB base rate above plus an assumed premium for dedicated Atlas Search Node compute (our estimate — no published per-op vector rate). mongodb.com/products/platform/atlas-vector-search
AlloyDB$4/ops-sec/yr
On-demand HA instance (8 vCPU / 64 GB, ~$2.49/hr) annualized, ÷ an assumed ~6,000 sustained ops/sec (our sizing assumption). cloud.google.com/alloydb/pricing
Couchbase$1/ops-sec/yr
Capella Enterprise tier, ~$0.56/node-hour, 3-node cluster, annualized, ÷ an assumed ~5,000 sustained ops/sec per node (our sizing assumption). Entered independently of any competitor figure. couchbase.com/pricing

Honest caveat: the consumption-priced rows (Firebase, MongoDB, DynamoDB) are solid — those vendors publish an actual $/operation rate, so the math is direct. The node/core-licensed rows (Oracle, SQL Server, AlloyDB, Couchbase) depend on a throughput-per-unit assumption we made up from general sizing heuristics, not a vendor spec sheet — swap in real benchmark data if/when we have it. Redis, Memcached, and Postgres have no fixed answer at all since cost scales with tier/instance choice; treat those three as directional floors, not commitments.
